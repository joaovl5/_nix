## Critical Issues

### C1. Circular import: `docker_runtime` ↔ `image_assets`

`image_assets.py:9` imports `docker_runtime` to raise `DockerRuntimeError`. `docker_runtime.py:14` imports `ImageAssets` from `image_assets`. This works today only due to Python's lazy module loading — any import-order change or top-level reference addition will cause `ImportError` at startup.

**Fix:** Extract `DockerRuntimeError` into `frag/exceptions.py`.

### C2. Duplicated slugify logic — silent divergence risk

`docker_runtime.py:38-44` (`_slugify_profile_name`) and `profiles.py:241-247` (`volume_name_for_profile`) implement the same regex normalization independently. If one changes without the other, container names and volume names diverge, silently breaking profile lookups.

**Fix:** Define once in `profiles.py` (canonical module for profile identity), import in `docker_runtime.py`.

### C3. Duplicated Docker invocation with different error contracts

- `docker_runtime.py:51-62` — returns `CompletedProcess`, caller checks returncode
- `profiles.py:81-88` — uses `check=True`, wraps in `DockerBackendError`

Two separate error hierarchies (`DockerRuntimeError` vs `DockerBackendError`) for the same Docker CLI. Changes to error handling must be replicated in both places.

**Fix:** Shared `frag._modules.docker_invoke` with configurable error behavior.

### C4. Duplicated environment variable constants

`bootstrap.py:8-11` and `docker_runtime.py:17-20` define identical env var names (`FRAG_BOOTSTRAP_TOKEN`, `FRAG_TARGET_UID`, `FRAG_TARGET_GID`, `FRAG_TARGET_SUPPLEMENTARY_GIDS`). A rename in one silently breaks the host-container contract.

**Fix:** Define once in a shared constants module.

### C5. Dangerous global monkeypatch in tests

`test_bootstrap.py:491-506` replaces `Path.read_text` for ALL `Path` instances globally. The lambda raises `KeyError` for paths not in its dict, bypassing `except OSError` guards in production code (`bootstrap.py:195`).

**Fix:** Create real files at `tmp_path` or mock `_container_root_matches_requested_owner` directly.

### C6. Full NixOS evaluation on every system rebuild

`images.nix:26-33` evaluates an entire NixOS system configuration via `import "${pkgs.path}/nixos" { ... }`. Since `users/lav.nix:10` imports `packages` eagerly, this runs on every `nixos-rebuild` / `home-manager switch` / `nix flake check`, adding 5-15+ seconds.

**Fix:** Gate behind a separate flake output (e.g. `packages.x86_64-linux.frag-image`).

### C7. Duplicate Python derivation with divergent dependencies

`images.nix:7-24` (`runtimeFrag`) and `default.nix:7-28` (`frag_runtime`) build the same source with different dependency sets. `runtimeFrag` omits `rich` — will crash if the code ever imports it. Two parallel dependency lists to maintain is fragile.

**Fix:** Build once with union of dependencies, or derive via `frag_runtime.override { ... }`.

### C8. `specialArgs.pkgs` shadows NixOS module system `pkgs`

`images.nix:28` injects outer `pkgs` into all module arguments, overriding the NixOS-configured `pkgs`. Works by accident today but will silently break with overlays or `nixpkgs.config`.

**Fix:** Remove `specialArgs`, pass `runtimeFrag` via a custom option.

### C9. Design spec files missing from HEAD

`docs/superpowers/specs/2026-04-17-frag-nixos-runtime-design.md` and `docs/superpowers/plans/2026-04-17-frag-nixos-runtime-redesign.md` were committed in `8beba07` but are not present at expected paths. Two critical design documents are unreachable.

## Architecture Assessment

### Strengths

- **Protocol-based abstractions** — `DockerBackend` and `ImageAssets` protocols enable testing without Docker
- **Sophisticated identity overlay** — nss_wrapper + setpriv handles both userns and normal cases
- **Token-based bootstrap handshake** — clean startup synchronization between host and container
- **`shared_assets_identity` hash** — prevents stale asset detection across updates
- **Nix→Python integration via `catalog.json`** — Nix builds tarball + catalog, Python discovers at runtime by walking up from binary location
- **Frozen dataclasses throughout** — immutable data structures prevent accidental mutation
- **Schema versioning** — explicit v2 with v1 refusal

### Weaknesses

- **DRY violations are the systemic issue** — 4 critical duplications + 10+ minor. Constants, slugify logic, Docker invocation, path canonicalization, bootstrap status paths, and shared asset mappings all live in multiple places.
- **Private API leakage** — `cli.py` reaches into `docker_runtime._current_supplementary_gids`
- **No cleanup path for partial failures** — orphaned containers on bootstrap timeout
- **No error-type dispatch** — error classification relies on substring matching of message strings

### Data flow

```
Nix build time                          Runtime (Python CLI)
========================                ====================

runtime-system.nix ──┐
                     ├─► images.nix ──► runtimeRootfs.tar
                     │                     │
image_catalog.nix ───┤                     │  docker import
                     │                     ▼
shared_assets ───────┤                Docker Image
                     │                     │
default.nix ─────────┘                     │  frag enter
     │                                     ▼
     ▼                                bootstrap.py
  symlinkJoin                      (inside container)
     │                                 │
     ▼                                 ▼
  $out/bin/frag ──► cli.py ──► docker_runtime.py ──► docker exec
```

### Evolution concerns

- **Adding new image types:** Well-designed — add entry to `images.nix`, catalog auto-generates. `ImageAssets` protocol is clean and extensible.
- **Adding new profiles:** Fully data-driven, no code changes needed.
- **Changing Docker runtime:** `DockerBackend` protocol abstracts profile operations, but `docker_runtime.py` calls `docker` CLI directly. Swapping to Podman or Docker SDK requires significant changes.
- **Hardcoded container paths:** `/workspace-root`, `/state/profile`, `/state/shared`, `/home/agent` are scattered across bootstrap.py, docker_runtime.py, and images.nix — not centralized.

### Operational concerns

- **Docker not running:** `FileNotFoundError` is caught cleanly, but daemon-not-running produces a raw Docker error message that isn't user-friendly.
- **Corrupted images:** `load_image` validates `image_ref` matches catalog, but no integrity check (checksum) on the tarball itself.
- **No disk space check** before `docker import` of the full NixOS tarball.
- **Bootstrap timeout is hardcoded** at 10 seconds (`docker_runtime.py:643`).
